HTML

Press F12 to view the html code for this page

  • Basic HTML Format:
    1. Document type declaration for browser
    2. HTML = Where all of the code will go and should also set the language here for search engine optimization and accessibility
      1. Head = Page data not rendered for user
        • Metadata
        • Links to external resources and scripts
        • Title for webpage, important for accessibility and search engine optimization
      2. Body = All user visible content
        1. Header
        2. Main
        3. Footer
  • You can create notes in HTML with <!-- Comments in here -->
  • <!DOCTYPE_html> should declare the HTML document at the very first line on top
  • <meta charset="UTF-8" /> is a character set containing pretty much all character for any language, recommended in regular use for HTML5
  • The head may contain <meta> tags like <charset> as well as <title> for the name of the webpage, <base> for defining a base URL, <link> to link to external resources, <script> for linking to scripts you want to use on the webpage, etc.
  • <link rel="stylesheet" href="directory or URL here" /> is used to link to a .css style sheet
     a directory of a file in a folder relative to where the html document is may look like "./assets/style.css"
  • <h1>Heading Here</h1> is used for headings, where #1 can be 1-6 and should be only nested in order. This is important for accessibility in screen readers and SEO (search engine optimization)
  • <p id="name"> can be used to create an id for this individual element to apply styles in CSS, like a paragraph element here
  • <p class="name"> can be used to set an element as a class, which can be used to identify multiple elements, add class="name" to another element to apply a style to all at once in CSS, the other elements wouldn't have to be paragraphs necessarily
  • <a href="URL" target="#1-#4">text on screen</a> can be used to create a hyperlink with text as the clickable element, where target can be...
    1. _self = default target value, opens the link in the same window/tab as it was clicked
    2. _blank = opens in a new window or tab
    3. _parent = opens the doceument in the parent frame
    4. _top = opens the document in the full body of the window
  • To create an image hyperlink instead use the following, where #1-#4 are as above
    <a href="URL" target="#1-#4">
     <img
      src="./assets/img.png" **OR** src="URLhere"
      alt="This images description"
      title="Will display when hovered over"
     /></a>
  • To create spacing you can add these character
    • &nbsp; = | |
    • &ensp; = | |
    • &emsp; = | |
  • <ol>List</ol> can be used to create an ordered list
  • <ul>List</ul> can be used to create an unordered list (bulleted list)
  • <li>List item</li> is how to insert a list element into an ordered or unordered list
  • Lists can be nested inside of each other to create subsections, you should nest a list inside of another list element
    <li>
     <ol>
      <li>List item</li>
     </ol>
    </li>
  • You can show html code formats in text form by interrupting it with a comment inline with the code or showing the <, text, and > putting a <q> before the element name and </q> after it inside the <> brackets without the quotation marks

Vocabulary

  • Head - Section containing data for the page that is not rendered for the user to see
  • Body - Main section of the html code, this is what's rendered to be shown to the user
  • Header - Top section of the page, where stuff at the very top of the page will be placed (ex. page banner or title like a company name)
  • Main - The majority section of the page where all of the main elements and content will go
  • Footer - Bottom section of the page, where stuff at the very bottom of the page will be placed (ex. copyright information, social media links, website section links like "contact us", "help", or other popular/important areas on the site)
  • Heading - An element on the webpage used to identify a section of content coming up, like the title for an individual chapter of a book
  • Hyperlink - A section of text or other element like an image, that when clicked directs you to a URL embedded in it
  • Title - Used to create the text that shows when hovering over an element or link and important to search engine optimization
  • SEO - Search engine optimization
  • Metadata - Different types of data the browser will use to get resources for how to display the page, always found inside the head

Vocabulary

  • CSS - Cascading Style Sheets
  • Style Sheet - Either a section at the top of the .html doc or usually a separate .css doc containing all of the styling for a webpage
  • Styles - Parameters assigned to HTML elements to give them sizes, colors, spacing, margins, backgrounds and other visual or relative content changes/additions
  • Class - An identifier that can be used to access and change content in multiple elements at once
  • ID - An identifier that can be used to access and change content in an individual element and no more, this is unique and cannot be duplicated
  • Box Model - The way we can describe and visualize the space around every HTML element on a page, from inside to outside it consists of the content, padding, border, and margin
  • Content - The innermost area of each HTML element, what is actually visible and rendered for the user (excluding background colors etc.)
  • Padding - The area directly outside of the content inside of an HTML element
  • Border - The area separating the padding and margin
  • Margin - The outermost section of a HTML element, be careful of collapsing margins when separate elements margins are able to touch

Git

  • Git is used to keep track of versions as you save changes, add features, and fix bugs. You can use it to revert to old versions and keep your live project or service safe from errant changes while modifying stuff in the background
  • GIT FLOW: (in GitBash or other console use command inside of parenthesis at given step)
     **USE (git pull origin main) OFTEN, AT LEAST DAILY WHEN WORKING ON REPOSITORY, main MAY BE ANOTHER BRANCH NAME LIKE MASTER**
    1. Clone the repository on GitHub (preferably using SSH cloning from the coding dropdown menu)
    2. Open an issue for fixing a bug or adding feature on GitHub
    3. Create a branch (-b) and move to it (checkout) to have a safe section to work in (git checkout -b branchtype/branchname)
    4. Work on your code!
    5. Stage for commit, to stage all changes use (git add -A)
    6. Commit work and either continue work back to step 4 or move on to the next step (git commit -m "your commit message here")
    7. If ready to prepare to push, first do a git pull to check version (git pull origin main)
    8. Now push your code to GitHub (git push origin branchtype/branchname)
    9. Create a pull request on GitHub and collaborate or fix any issues
    10. If ready to merge then either merge on GitHub after pull request is approved or first (git checkout) to the base branch you want to merge into in your terminal and insert the comparison branch in "branchtype/branchname" when you use (git merge branchtype/branchname)
    11. It's possible you may now want to delete the feature branch you were working in and move on to a new feature or bugfix branch
  • git status: checks what branch we are currently on
  • git remote -v: check the fetch and push address you have currently, can use before git pull or push to verify where you are directed

Vocabulary

  • Git - A free open source version control tool that is installed locally on a computer
  • GitHub - A paid service to allow Git to run in the cloud connection multiple developers or computers
  • Default Branch - Is the primary branch of a project, usually called main or master
  • Feature Branch - Is the working branch where you will typically make changes before merging back higher up in the chain
  • Base Branch - The higher level branch you are merging into on a merge request
  • Comparison Branch - The branch you are merging back into the main code on a merge request
  • Staging - The process of lining up changes made to commit into Git
  • Commit - When you save changes in a branch that are logged in Git
  • Pull Request - A submission to merge the commits made in a branch being worked on (feature branch) to the base branch, must be approved and notes or changes may be made at this point
  • Merge/Merging - The process in which a feature branch with changes made is added back into, or merged, with the base branch it came from
