HTML

Press F12 to view the html code for this page


  • Basic HTML Format:
    • Declerations like document type
    • HTML = Where all of the code will go and should also set the language here for search engine optimization and accessibility
      • Head = Page data not rendered for user
        • Metadata
        • Links to external resources and scripts
        • Title for webpage, important for accessibility and search engine optimization
      • Body = All user visible content
        • Header
        • Main
        • Footer
  • To create comments in HTML you need to use <!-- Comments in here -->
  • You should declare the html document at the top with <!DOCTYPE_html>
  • The head element contains information and metadata about the webpage, and is not normally visible to users.
  • The head may contain <meta> tags as well as <title>, <base>, <link>, <script>, etc.
  • To link to a .css style sheet use <link rel="stylesheet" href="directory or URL here" />
     a directory of a file in a folder relative to where the html document is may look like "./assets/style.css"
  • The body element represents the content the user will be shown.
  • Inside the body element are usually a headings <h#> </h#> where # can be 1-6 and should be only nested in order. This is important for accessibility in screen readers and SEO (search engine optimization)
  • You can uniquely identify elements, like paragraphs, ex. <p id="name">, but each element id must be unique
  • You can also create a class which can apply to multiple elements, ex. <p class="name">, then add class="name" to another element to apply a style to all at once in CSS
  • To create spacing you can add these character
    • &nbsp; = | |
    • &ensp; = | |
    • &emsp; = | |
  • You can show html code formats in text form by interrupting it with a comment inline or showing the <, text, and > putting a <q> before the element name and </q> after it inside the <> brackets without the quotation marks

CSS


  • Usually using inline styles is not good, creating a seperate .css style sheet is good practice
  • Creating global values can be done inside of * {} bracket
  • You can creat comments in CSS with /* Comments in here */
  • To apply styling to multiple elements, classes, ids, or items use commas to separate them before the opening bracket like this,
     h1, .classname, #idname {
     }

Git


  • GIT FLOW: (in GitBash or other console use command inside of parenthesis at given step)
    • **USE (git pull origin main) OFTEN, AT LEAST DAILY WHEN WORKING ON REPOSITORY, main MAY BE ANOTHER BRANCH NAME LIKE MASTER**
    1. Clone the repository on GiHub (preferably using SSH cloning from the coding dropdown menu)
    2. Open an issue for fixing a bug or adding feature on GitHub
    3. Create a branch (-b) and move to it (checkout) to have a safe section to work in (git checkout -b branchtype/branchname)
    4. Work on your code!
    5. Stage for commit, to stage all changes use (git add -A)
    6. Commit work and either continue work back to step 4 or move on to the next step (git commit -m "your commit message here")
    7. If ready to prepare to push, first do a git pull to check version (git pull origin main)
    8. Now push your code to GitHub (git push origin branchtype/branchname)
    9. Create a pull request on GitHub and collaborate or fix any issues
    10. If ready to merge then either merge on GitHub or first checkout to the
  • git status: checks what branch we are currently on
  • git checkout -b branch-type/name: creates and switches to a new branch ex. "feature/menu-buttons"
